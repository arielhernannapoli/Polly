 a nivel global?"

✅ RESPUESTA: SÍ, hay 5 formas diferentes

✅ OPCIÓN 1: EXTENSIÓN SIMPLE (RECOMENDADA E IMPLEMENTADA)

   Ubicación: Extensions/HttpClientExtensions.cs (en cada proyecto)
   
   Uso: .AddCustomResilienceHandler()
   
   Ventajas:
   • Código limpio (2 líneas vs 50+ antes)
   • Centralizado
   • Fácil de mantener
   • Escalable


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

De: Polly.Extensions.Http (3.0.0) ⚠️ CON VULNERABILIDADES
A:  Microsoft.Extensions.Http.Resilience (10.4.0) ✅ SEGURO


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💻 CÓMO USAR EN TUS 3 PROYECTOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROYECTO 1: Polly.Retry.Example
─────────────────────────────────
builder.Services.AddHttpClient("BackendApi")
    .AddCustomResilienceHandler();


PROYECTO 2: Polly.Retry.Example.TypedClient
─────────────────────────────────────────────
builder.Services.AddHttpClient<IBackendService, BackendService>()
    .AddCustomResilienceHandler();


PROYECTO 3: Polly.Retry.Example.DelegatingHandler
───────────────────────────────────────────────────
builder.Services.AddHttpClient<IBackendService, BackendService>(client =>
{
    client.BaseAddress = new Uri("https://localhost:7056");
})
.AddCustomResilienceHandler();

    for (var name in clientNames) { ... }

    .AddHttpMessageHandler<LoggingHandler>()
    .AddCustomResilienceHandler()

✅ RETRY (Reintento)
   • 3 intentos
   • Backoff exponencial (1s → 2s → 4s)
   • Jitter automático

✅ CIRCUIT BREAKER (Corta-circuitos)
   • Se abre si 50% de fallos en 30 segundos
   • Se cierra automáticamente

✅ TIMEOUT (Límite de tiempo)
   • 10 segundos por request
