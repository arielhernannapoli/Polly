PREGUNTA:
"¿Puede aplicarse el AddStandardResilienceHandler a todos los httpclient 
 a nivel global?"

RESPUESTA:

✅ SÍ, hay 5 formas diferentes:

1️⃣  EXTENSIÓN SIMPLE (IMPLEMENTADA - RECOMENDADA)
    • Uso: .AddCustomResilienceHandler()
    • Centralizado en Extensions/HttpClientExtensions.cs
    • Perfecto para 1-10 clientes

2️⃣  DIRECTO
    • Uso: .AddStandardResilienceHandler()
    • Sin extensión, más simple
    • Para casos triviales

3️⃣  LOOP + EXTENSIÓN
    • Registra múltiples clientes de una vez
    • Perfecto para 2-10 clientes similares

4️⃣  STACK DE HANDLERS
    • Combine: Logging + Auth + Resilience
    • Para lógica personalizada compleja

5️⃣  CONFIGURACIÓN DESDE JSON
    • appsettings.json + loop
    • Para producción con múltiples entornos
    • Dinámico, sin código hardcodeado

Ver GUIA_HANDLERS_GLOBALES.md para ejemplos completos de cada opción.

PROYECTO 3: Polly.Retry.Example.DelegatingHandler

    using Polly.Retry.Example.DelegatingHandler.Extensions;

        client.BaseAddress = new Uri("https://localhost:7056");
    })
    .AddCustomResilienceHandler();

═══════════════════════════════════════════════════════════════════════════════════
                          💡 NOTAS IMPORTANTES
═══════════════════════════════════════════════════════════════════════════════════

⚠️  Los métodos AddAggressiveResilienceHandler() y 
    AddConservativeResilienceHandler() son ALIAS que hacen lo mismo que
    AddCustomResilienceHandler() en esta implementación.
    
    Razón: Microsoft no expone parametrización en AddStandardResilienceHandler()
    
    Si necesitas diferentes configuraciones, usa DelegatingHandlers personalizados
    (ver EJEMPLOS_AVANZADOS.cs)

✅  El DelegatingHandler personalizado (PollyDelegatingHandler.cs) fue eliminado
    porque ahora Microsoft maneja la resiliencia automáticamente.
    
    Si necesitas lógica custom, crea un nuevo DelegatingHandler que combine
    con .AddStandardResilienceHandler()

✅  La migración es 100% compatible con .NET 8 y no rompe nada.

                        📞 PREGUNTAS Y RESPUESTAS
═══════════════════════════════════════════════════════════════════════════════════

P: ¿Necesito cambiar algo en mis servicios que usan HttpClient?
R: No, todo es transparente. El HttpClient funciona igual, solo con mejor
   resiliencia internamente.

P: ¿Cómo sé que se aplica realmente?
R: Agrega logging temporal para verificar (ver EJEMPLOS_AVANZADOS.cs)

P: ¿Qué pasa si tengo 100 clientes?
R: Usa configuración desde JSON (OPCIÓN 5 en GUIA_HANDLERS_GLOBALES.md)

P: ¿Puedo volver a Polly.Extensions.Http?
R: No recomendado, tiene vulnerabilidades. Mejor extender con handlers custom.

P: ¿Funciona con .NET 7?
R: No, es solo para .NET 8+. Para .NET 7, usa Polly directamente.
